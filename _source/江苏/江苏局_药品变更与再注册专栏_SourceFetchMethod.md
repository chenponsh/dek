## 页面特点

这些江苏局栏目页不是普通静态列表：

- 栏目页 HTML 中内嵌 `datastore` / `recordset` 片段，列表记录包含文章链接、标题、发布日期。
- PC 端和移动端可能各有一份列表记录，抓取时需要按文章 URL 去重。
- `curl`、Python `requests` 等普通请求容易拿到 403 / 加速乐提示页；已验证可用 PowerShell `Invoke-WebRequest` 读取 `RawContentStream` 取得真实 HTML。
- 单篇文章正文通常位于 `<!--ZJEG_RSS.content.begin-->` 与 `<!--ZJEG_RSS.content.end-->` 之间；页面也可能存在 PC / 移动端重复正文，解析第一段正文即可。

## 写回 Markdown 规则

保留 source 笔记 frontmatter，并在 frontmatter 与正文之间保留一个空行。

写回前按 `AGENTS.md` 中的来源笔记主题筛选规则过滤：本库以“化学药品制剂”为关键主题；如栏目文章混入与该主题无关的中药、化妆品、医疗器械等内容，不写入正式 source 内容。

每个使用本方法的 source 笔记正文先保留统一入口：

```markdown
## 整理规则

本来源使用江苏局官网专栏页共用整理规则：

- [[江苏局_药品变更与再注册专栏_SourceFetchMethod]]
```

其后写入来源数据表格：

```markdown
| 问题 | 解答 | 发布日期 |
| --- | --- | --- |
| ... | ... | 2026-05-29 |
```

处理细节：

- 正文段落换行转换为 `<br>`，避免破坏 Markdown 表格。
- 单元格中的竖线 `|` 需要转义为 `\|`。
- 不额外增加来源链接、序号等列。
- 不改写问答实质内容。

## 增量抓取策略

1. 读取 source 笔记 frontmatter 中的 `url`，确定要抓取的栏目页。
2. 解析栏目页列表记录，并按文章 URL 去重。
3. 与本地表格按“问题 + 发布日期”去重；官网存在、本地不存在的记录视为新增。
4. 新增记录按官网列表顺序插入表头下方，使 source 笔记保持官网默认倒序。
5. 若“问题 + 发布日期”已存在但正文不同，视为官网修订；匹配明确时更新对应行，匹配不明确时停止并提示人工核对。
6. 如栏目结构、分页结构或正文定位方式变化，应停止机械增量，先更新本方法笔记。
